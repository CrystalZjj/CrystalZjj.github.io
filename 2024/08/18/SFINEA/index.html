1. 前言

1.0 模板参数

模板参数可以显式指定，也可以通过模板参数推导来隐式确定。

1.1 模板参数推导

编译器根据 【函数调用的参数】 推断 【模板函数或类模板】 中的 【模板参数类型】。
推导过程不会再考虑显式指定的模板参数

1.2 显式指定模板参数

调用模板时，显式地指定模板参数。

1.3 非类型模板参数

c++11开始，模板不仅可以使用类型作为模板参数，还可以使用常量表达式作为非类型模板参数。这些表达式在模板实例化时进行替换和求值

2. 何谓SFNIAE？

SFINAE（substitution failure is not an error：替换失败不是一个错误）与模板元编程相关的规则；此规则适用于函数模板的重载决策期间，当显式指定或推导的类型替换模板参数失败时，【将丢弃重载集中的特化】，而非导致编译报错。

2.1 模板参数被替换的两个步骤

explicitly specified template arguments are substituted before template argument deduction
deduced arguments and the arguments obtained from the defaults are substituted after template argument deduction

即：

  • 显式指定的模板参数，在模板参数推断之前被替换
  • 推导的参数以及未被显式指定但有默认值的参数，在模板参数推断之后被替换

2.2 模板参数替换发生的场景

all types used in the function type (which includes return type and the types of all parameters)
all types used in the template parameter declarations
all types used in the template argument list of a partial specialization

  • 函数类型中的所有类型（包括所有返回值类型和所有参数类型）

  • 模板参数声明中的类型

  • 部分特化模板参数列表中的替换

all expressions used in the function type
all expressions used in a template parameter declaration
all expressions used in the template argument list of a partial specialization

  • 函数类型中使用的所有表达式

  • 模板参数声明中使用的所有表达式

  • 部分特化的模板参数列表中的所有表达式

4. SFINAE的实际应用场景

4.1 enable_if

4.1.1 enable_if是什么？

【基于条件】在【编译时】【启用或禁用】函数或类

4.1.2 enable_if如何用？

  • 作为函数模板参数的默认类型
    匹配成功解析：
    (1) 显式指定int类型调用模板时，模板参数声明类型发生模板参数推导；
    (2) std::is_integral<T>::value 的值为true，会匹配到偏特化类模板 enable_if<true, int>；
    (3) 此偏特化类模板有 type 类型， 因此匹配到合适的模板，模板实例化成功；

匹配失败解析：
(1) 显式指定float类型调用模板时，模板参数声明类型发生模板参数推导；
(2) std::is_integral<T>::value 的值为false，会匹配到偏特化类模板 enable_if<false, int>；
(3) 此偏特化模板没有 type 类型，因此匹配失败，但是编译器不会报错，而是忽略此模板，接着向下匹配；【SFINAE】的体现

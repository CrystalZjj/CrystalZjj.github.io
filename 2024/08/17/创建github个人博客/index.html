<folder>目录内会生成如下文件和文件夹；【如果不去主动开发themes，这些含义大致了解即可，使用者只需关注在哪里做改动】

文件 含义
scaffolds【文件夹】 当hexo创建新md文件时，会基于scaffolds内的文件来创建新md文件
source【文件夹】 放置网站内容的地方，hexo会忽略隐藏文件以及名称以_（下划线）为前缀的文件或文件夹，_posts文件夹除外。会渲染markdown文件和html文件并且将它们放到public文件夹，其他的文件会直接复制
themes【文件夹】 主题文件，hexo会根据theme文件产生一个静态网页
packages.json 依赖的渲染器
_config.json 修改网站的一些配置文件

三. 关于hexo工程的注意点【根据实际需要看即可，更详细的请参考官方文档】

3.1 layout 布局

  • hexo中默认有三种布局：帖子（post）、页面（page）和草稿（draft）
  • 不同布局有不同的路径：post（source/_posts），page（source），draft（source/_draft）
  • 如果不想使用themes中的布局，令_config.json文件中的 layout : false 即可

3.2 filename 文件名

  • 默认hexo使用post title作为文件名，可以改变 _config.json 文件中的 new_post_name 来改变默认文件名【具体怎么用还没试过】

3.3 drafts 草稿

  • 基于 draft 层初始化的posts 被保存在 source/_drafts 文件夹；
  • 可以使用 publish 来将 草稿 drafts 移动到 source/_drafts 文件夹；
  • drafts 默认不显示，当执行hexo命令时可以添加 -option 选项 或者 配置_config.json文件中的 render_drafts参数 来显示

3.4 scaffolds

当运行此命令，hexo会试图去寻找在 scaffolds 文件夹中的 photo.md 文件，并且基于它创建 My gallary.md文件

3.5 支持的格式

  • hexo默认安装 markdown 和 ejs插件，即（hexo-rendered-marked 和 hexo-rendered-ejs）

3.6 Asset folders

更多使用细节请去参考：https://hexo.io/docs/asset-folders

  • Asset文件夹在source文件夹中是非post文件，例如有images、css或者javascripts；如果在hexo工程中要显示一些图片， 最简单的方法就是将这些图片放置到 source/images 文件夹下，然后可以使用下述命令引用图片
